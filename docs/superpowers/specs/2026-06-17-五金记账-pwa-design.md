# 五金记账 PWA 设计规格

## 1. 项目概述

家庭内部使用的五金记账 PWA 应用。妈妈在 iPhone 上通过语音输入记录客户拿货情况，爸爸在电脑上查看、编辑和管理记录。

## 2. 用户角色

| 角色 | 设备 | 主要操作 |
|------|------|----------|
| 妈妈 | iPhone | 语音输入商品名，手动填数量，选择客户 |
| 爸爸 | PC | 查看今日记录，编辑修改，查看历史，未来做报表分析 |

## 3. 核心流程

### 3.1 iPhone 端（妈妈）

```
打开 PWA → 客户列表（大按钮 + 搜索栏）
→ 搜索/点击某客户 → 该客户的记录页
→ 点击"新建记录" → 添加商品（语音输入名称 + 手动填数量）
→ 保存（自动记录当前时间）
→ 同一客户可多次新建记录（不同时间段）
```

### 3.2 PC 端（爸爸）

```
打开 PWA → 今日记录概览（默认显示今天）
→ 日期选择器可切换查看历史
→ 客户卡片列表，每张卡片显示：客户名 + 商品清单
→ 可编辑/删除/新增记录条目
→ 未来预留：图表报表
```

## 4. 数据模型

### customers（客户表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| name | text | 客户名称 |
| phone | text | 电话（可选） |
| created_at | timestamp | 创建时间 |

### records（记录表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| customer_id | uuid | 关联客户 |
| date | date | 日期（方便按天查询） |
| created_at | timestamp | 精确时间（自动记录） |
| updated_at | timestamp | 更新时间 |

### record_items（记录明细表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| record_id | uuid | 关联记录 |
| product_name | text | 商品名称（语音输入） |
| quantity | numeric | 数量 |
| created_at | timestamp | 创建时间 |

### 关系
- customers 1:N records（一个客户可有多条记录）
- records 1:N record_items（一条记录可包含多个商品）

## 5. 技术方案

| 技术 | 用途 |
|------|------|
| HTML/CSS/JS | 前端（无框架，GitHub Pages 静态部署） |
| Supabase JS SDK | 数据库访问 + 实时同步 |
| Web Speech API | 语音识别（iPhone Safari 原生支持） |
| PWA (manifest + SW) | 可安装到主屏幕，离线缓存 |

## 6. 界面设计

### 6.1 iPhone 端
- 客户列表：大按钮（最小 48px 触摸区域），搜索栏支持语音+键盘
- 记录页：今日记录列表，每条记录展开显示商品明细
- 添加商品：底部弹出面板，商品名输入框带麦克风图标，数量用数字键盘
- 整体风格：大字体、大按钮、高对比度、减少操作步骤

### 6.2 PC 端
- 顶部：日期选择器（默认今天）+ 搜索
- 主区域：客户卡片网格布局
- 卡片：客户名标题 + 商品表格（商品名、数量、时间）
- 编辑：点击商品行可编辑，支持删除和新增
- 响应式：手机端自动切换到 iPhone 布局

## 7. 部署方案

- GitHub Pages 托管静态文件
- Supabase 托管数据库
- 无需服务器，零运维成本
